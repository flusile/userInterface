Damit der Client (html, js) ordentlich mit dem Server reden kann, hier ein paar Vereinbarungen:

Felder, die initial beim ersten Laden der Seite gefüllt werden sollen, müssen die Form
{{ name[:subname] }} 
haben. Dabei kann es beliebig viele Subnamen geben: z.B. {{ name:subname:subsub:subsubsub }}
Diese Namen werden im Server beim Ausliefern der Seite durch die entsprechenden Werte aus dem laufenden Programm ersetzt.
Sie müssen daher auch immer zu den im Server bekannten namen passen.

Felder, die später vom Server aktualisiert werden sollen, müssen in ihrem class-Attribut einen Namen haben.
Dieser Name entspricht ggf. dem oben genannten Platzhalter, wobei jedoch alle : durch _ ersetzt werden.
Da solche class-Attribute nur an Tags vergeben werden können, sind diese Felder in einzelnen <span>..</span> Tags untergebracht.

Allerdings erfolgt über die class-Attribute auch die Markierung für css-Formate. Hierfür gibt es zwei Möglichkeiten, 
die wir ausprobieren müssen:
1. Man kann dem class-Attribut eine Liste von Namen zuweisen. 
   Dann werden alle damit verbundenen css-Formatierungen angewendet.
   Das kann ggf. dzu führen, daß Formatierungen in übergeordneten Ebenen nicht wirken (--> Test)
